# Observability

Use this directory for shared observability standards, dashboards, logging approaches, and SLO thinking.

## Current State

Observability appears to be handled ad hoc today rather than through a dedicated shared stack.

## Recommended Additions

- Prometheus for metrics
- Grafana for dashboards
- Loki for logs

## What Good Looks Like

- each service has basic uptime and resource visibility
- ingress and host issues can be traced quickly
- backup jobs and critical services are observable
